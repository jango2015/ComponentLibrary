
一.目录说明：

    组件所在的项目在lib目录下，测试该组件的Demo在web目录下，mywork.sln是解决方案文件。
与组件相关的文件有两部分：
1.程序集：NetFocus.Components.SearchComponent.dll
2.一个组件运行所必须的文件夹：其默认名称为Data，见web目录下的Data文件夹


二.运行解决方案文件mywork.sln的前提条件：

    先对web目录新建一个虚拟目录，虚拟目录的名称是mywork,
建好虚拟目录后就可以直接运行解决方案文件查看并运行代码了。


三.从头使用本组件的步骤：

    1）新建一个Asp.Net Web应用程序。添加对组件NetFocus.Components.SearchComponent.dll的引用。
添加引用时有两种方式：1.如果只是在解决方案资源管理器中为项目添加引用，则在工具
箱中不会出现该组件，也就是说，不能直接从工具箱中把组件拖到Web窗体上；2.如果直接在工具
箱中添加新的组件，则可以直接从工具箱中把组件拖到Web窗体上；但是，不管是用哪种方式添加
引用，都需要到aspx页面的HTML视图去为这个组件添加一个属性DataPath属性，因为现在组件还
不支持直接在属性面板中设置DataPath属性，具体原因我也不清楚。

    2）将web目录下的Data文件夹拷贝到你刚建的Web应用程序的根虚拟所对应的目录目录下，该文件夹的名称
你可以自己修改，只要能保证修改后的名称与DataPath属性对应即可。千万要注意：如果还没有理解Data文件夹
中的文件的含义，千万不要随便修改或移动它们的位置，以防组件不能正常运行。

    3）组件有四种状态，每一种状态完成一个单独的功能。在状态之间进行切换不是自动的，必须通过使用者来
触发。具体的切换方式见Demo项目。当然，你可以按照不同的需要，在不同的页面中都使用这个组件，只要将其
显示的状态设置为不同的值即可。
